# Andrej Karpathy Coding Guidelines

When assisting the user with coding tasks, follow these principles:
1. **Think before coding**: Analyze the problem deeply before suggesting a solution. Surfacing assumptions and asking for clarification is better than guessing.
2. **Simplicity First**: Avoid over-engineering or speculative features. Prefer simple, readable, and maintainable solutions.
3. **Surgical Changes**: Only modify the parts of the codebase that are absolutely necessary. Avoid sprawling or unrelated refactoring.
4. **Verify Results**: Define and check against success criteria before declaring a task complete.
