# Hotel Pousada Delplata Design System

Design system estratégico para um site de hotelaria premium com foco em reserva direta.

Posicionamento:

`Family Retreat + Boutique Hotel + Editorial Luxury`

Essência:

Um lugar onde famílias desaceleram, se reconectam e vivem Serra Negra com conforto, natureza e hospitalidade autêntica.

---

## 1. Brand Direction

### Brand Promise

Hotel Pousada Delplata é o refúgio de fim de semana e férias curtas para famílias que querem descanso real, clima de montanha e momentos juntos, sem excesso, sem pressa e sem estética de portal de reservas.

### Emotional Territory

- tranquilidade
- natureza
- acolhimento
- memória em família
- leveza
- tempo desacelerado
- autenticidade
- sofisticação discreta

### What The Experience Should Feel Like

- silêncio confortável
- manhã com luz natural
- café sem pressa
- crianças aproveitando com segurança
- pais descansando de verdade
- Serra Negra como atmosfera, não como clichê turístico

### What This Brand Is Not

- não é resort
- não é OTA
- não é portal turístico
- não é “luxo ostentação”
- não é pousada rústica improvisada
- não é hotel business

---

## 2. Creative Positioning

### Reference Blend

O sistema visual deve se posicionar entre:

- `Aman Resorts`: silêncio visual, sofisticação e respiro
- `The Hoxton`: proximidade e calor humano
- `Design Hotels`: narrativa visual e curadoria
- `Fasano`: elegância contida
- `Airbnb`: fotografia aspiracional e sensação de viver o lugar
- `Jardim Monte Verde`: clima de refúgio com natureza

### Core Message

Não vender apenas hospedagem.

Vender a sensação de:

`“Nossa família precisa disso neste fim de semana.”`

---

## 3. Design Principles

### 3.1 Photography Before UI

A interface deve sempre servir a fotografia.

- imagens amplas
- pouco ruído visual
- texto com contraste suficiente, mas sem competir com a foto

### 3.2 White Space Is Mandatory

Respiro é parte da percepção de luxo.

- evitar blocos comprimidos
- evitar muitas divisões rígidas
- permitir leitura lenta

### 3.3 One Message Per Section

Cada seção deve comunicar uma única ideia central.

Exemplos:

- hero: desejo
- sobre: acolhimento
- acomodações: escolha
- experiências: lifestyle
- reviews: confiança

### 3.4 Fewer Cards

Evitar estética de dashboard.

- menos grids excessivos
- menos caixas repetidas
- mais blocos editoriais

### 3.5 Emotional Selling First

Antes da reserva, o usuário precisa se imaginar no local.

### 3.6 Direct Booking With Premium Simplicity

O fluxo de reserva deve ser:

- imediato
- claro
- elegante
- sem complexidade desnecessária

## 4. Typography System

Usar somente duas famílias tipográficas.

### 4.1 Display Font

`Libre Baskerville`

Uso:

- hero titles
- headlines editoriais
- momentos de sofisticação

Pesos permitidos:

- `400`
- `700`

Nunca usar para:

- datas
- números de hóspedes
- labels de formulário
- navegação
- interface funcional

### 4.2 UI Font

`Inter`

Uso:

- navegação
- botões
- formulários
- corpo de texto
- avaliações
- motor de reserva

Pesos permitidos:

- `400`
- `500`
- `600`
- `700`

### 4.3 Typographic Roles

#### Hero Display

- família: `Libre Baskerville`
- peso: `400`
- tamanho: `56px` a `92px`
- line-height: `0.9` a `0.98`
- tracking: levemente negativo ou neutro

#### Editorial Headline

- família: `Libre Baskerville`
- peso: `400`
- tamanho: `40px` a `64px`
- line-height: `1.0`

#### Section Heading

- família: `Inter`
- peso: `600`
- tamanho: `28px` a `44px`
- line-height: `1.1` a `1.2`

#### Body Large

- família: `Inter`
- peso: `400`
- tamanho: `20px` a `24px`
- line-height: `1.6` a `1.8`

#### Body Default

- família: `Inter`
- peso: `400`
- tamanho: `16px` a `18px`
- line-height: `1.7`

#### UI Label

- família: `Inter`
- peso: `500` ou `600`
- tamanho: `11px` a `13px`
- uppercase
- tracking: `0.12em` a `0.24em`

## 5. Color System

### 5.1 Core Palette

| Token | Valor | Papel |
|---|---|---|
| `forest` | `#283223` | cor principal da marca |
| `gold` | `#BBB863` | acento premium natural |
| `cream` | `#F5F5F5` | fundo principal |
| `white` | `#FFFFFF` | respiro, contraste e áreas leves |
| `black` | `#090909` | contraste profundo e hero dark |
| `whatsapp` | `#00E676` | CTA flutuante WhatsApp |

### 5.2 Color Philosophy

- sem cores vibrantes
- sem gradientes agressivos
- sem azuis saturados
- sem roxos
- dourado sempre contido
- verde sempre sofisticado e natural

### 5.3 Supporting Neutrals

Recomendados para sistema:

| Token | Sugestão |
|---|---|
| `ink-soft` | `rgba(9, 9, 9, 0.82)` |
| `forest-soft` | `rgba(40, 50, 35, 0.72)` |
| `gold-soft` | `rgba(187, 184, 99, 0.18)` |
| `line-light` | `rgba(255, 255, 255, 0.16)` |
| `line-dark` | `rgba(40, 50, 35, 0.12)` |

### 5.4 Contrast Rules

- texto sobre imagem deve ser branco ou cream com sombra suave
- texto funcional deve priorizar legibilidade, nunca estilo acima de contraste
- gold deve ser usado como detalhe, não como texto principal em áreas de baixa leitura

## 6. Shape System

Usar somente três raios:

- `8px`
- `16px`
- `24px`

### Usage

- `8px`
  - pequenos controles
  - chips discretos
  - botões compactos

- `16px`
  - inputs
  - cards pequenos
  - popovers

- `24px`
  - containers premium
  - booking bar
  - painéis editoriais

Não usar:

- raios aleatórios
- mistura excessiva de cantos diferentes na mesma seção

---

## 7. Spacing System

### Section Spacing

- `Small Section`: `120px`
- `Medium Section`: `160px`
- `Large Section`: `220px`

### Internal Spacing

Recomendação operacional:

- `8px`
- `12px`
- `16px`
- `24px`
- `32px`
- `48px`
- `64px`

### Layout Philosophy

O site deve respirar como editorial premium.

- margens amplas
- poucas colunas simultâneas
- evitar sensação de “bloco sobre bloco”

---

## 8. Layout System

### 8.1 Site Width

- estrutura fluida com container central
- conteúdo textual ideal entre `680px` e `820px`
- blocos de imagem podem ocupar larguras maiores

### 8.2 Viewport Usage

Hero deve ocupar:

- `100vh`
- `100vw`

Sem:

- barras horizontais
- conteúdo vazando abaixo da dobra
- widgets grandes comprimindo o hero

### 8.3 Content Rhythm

Sequência ideal:

- imagem ou emoção
- headline
- supporting copy
- CTA

---

## 9. Hero System

### Objective

O hero não deve explicar tudo.

O hero deve criar desejo.

### Structure

- logo à esquerda
- navegação transparente e mínima
- imagem ou vídeo cinematográfico em tela cheia
- 1 headline
- 1 supporting sentence
- 2 CTAs no máximo

### Hero Copy Rules

Permitido:

- `Seu refúgio em Serra Negra`
- `Hospedagem para toda a família em Serra Negra`
- `Um lugar para desacelerar em família`

Não permitido:

- múltiplas promoções
- excesso de selos
- blocos de texto longos
- formulário grande no centro do hero

### Hero CTA Structure

- Primário: `Ver disponibilidade`
- Secundário: `Conheça a pousada`

### Hero Tone

- silencioso
- sofisticado
- emocional
- aspiracional

## 10. Social Proof

### Position

Logo após o hero.

### Layout

Horizontal, elegante, mínimo.

### Content

- `★★★★★ 4.9 Google`
- `★★★★★ 8.6 Booking`
- `★★★★☆ 4.3 Tripadvisor`
- `15+ anos de história`

### Rules

- sem cápsulas pesadas
- sem estética de review widget
- usar typography limpa
- deixar parecer curadoria, não plugin
- usar o componente oficial `EditorialStats` para qualquer faixa de métricas editoriais ou prova social
- não recriar esse padrão manualmente com classes locais fora do componente de sistema

## 11. Booking Experience

### Position

Pode ficar:

- na borda inferior do hero
- ou imediatamente após social proof

### Required Fields

- Check-in
- Check-out
- Guests
- Search

### Booking Bar Pattern

- barra horizontal escura, elegante e compacta sobre fundo claro
- quatro blocos principais: `Check-in`, `Check-out`, `Hóspedes`, `Buscar`
- labels pequenas em uppercase
- valores principais com leitura imediata
- divisórias sutis entre campos
- CTA areia/dourado claro com texto escuro e microcopy curta de tarifa
- hóspedes em linguagem humana, sem expor complexidade logo de início
- idade das crianças pode ficar dentro do seletor de hóspedes, não no corpo principal da barra

### Forbidden Fields

- cupom
- promoções no início da jornada
- campos demais
- ruído de portal de reservas

### Booking UI Tone

- premium
- simples
- convidativo
- sem parecer sistema legado

### UX Principles

- mostrar só o essencial
- datas com leitura clara
- hóspedes em linguagem humana
- CTA sempre visível

## 12. Homepage Architecture

### 1. Hero Fullscreen

Imagem dominante + mensagem emocional + CTAs.

### 2. Social Proof

Confiança imediata, sem exagero.

### 3. Reservation Engine

Fluxo direto e elegante.

### 4. About Delplata

Bloco editorial com imagem ampla e texto curto.

Mensagem:

- acolhimento
- clima de montanha
- memória em família

### 5. Accommodations

Máximo 2 ou 3 categorias.

Formato:

- imagens grandes
- pouco texto
- foco em estilo de hospedagem

### 6. Experiences

Curadoria emocional, não lista operacional.

Exemplos:

- piscina
- café da manhã
- natureza
- lazer em família

### 7. Gallery

Fotografia estilo Airbnb editorial.

- imagens grandes
- poucos elementos de UI
- sensação de desejo e vivência real

### 8. Reviews

Mistura de:

- Google
- Booking
- Tripadvisor
- falas curtas de hóspedes

### 9. Holiday Dates

Somente perto do fim da página.

Nunca antes de:

- Sobre
- Acomodações
- Experiências

Formato oficial:

- usar bloco editorial horizontal, não carrossel de cards genéricos
- selo discreto no topo com `UI Label`
- headline editorial à esquerda com `Libre Baskerville`
- imagem principal destacada à direita
- faixa branca inferior com até `3` datas clicáveis
- CTA final escuro integrado à seção

Tokens e linguagem:

- fundo principal em `cream`
- bloco final em `forest`
- acentos e ícones em `gold`
- textos funcionais em `Inter`
- títulos editoriais em `Libre Baskerville`
- usar apenas raios do sistema: `16px` e `24px`

Regras:

- não usar slider/carrossel como padrão principal nessa seção
- não empilhar múltiplos cards promocionais independentes
- a data selecionada deve atualizar imagem e CTA da seção
- o CTA deve apontar para a data ativa
- a seção deve parecer campanha editorial, não listagem operacional

### 10. Final CTA

Imagem emocional grande.

Headline recomendada:

`Pronto para sua próxima viagem?`

Botões:

- `Reservar Agora`
- `WhatsApp`

## 14. Component Guidelines

### Buttons

#### Primary

- fundo `forest`
- texto branco
- UI font `Inter`
- peso `600`
- raio `16px`
- hover com micro elevação (`-1px`)
- hover com sombra suave e perceptível
- hover nunca deve parecer agressivo ou “ecommerce”

#### Secondary

- fundo transparente ou cream
- borda suave
- texto `forest`
- quando aplicado sobre imagem ou fundo escuro, o hover deve ser perceptível com micro elevação, borda ligeiramente mais forte e preenchimento sutil
- evitar hover inexistente ou apenas imperceptível em CTAs com contorno

#### Hero CTA

- fundo `gold`
- texto `forest` ou `black`
- sensação de peça refinada, não botão agressivo
- hover com escurecimento muito leve ou redução sutil de brilho
- pode usar micro elevação desde que discreta

#### Dark Surface CTA

Uso:

- navbar transparente sobre hero
- CTA secundário sobre imagem
- ações sobre fundos escuros ou fotografia

Estilo:

- fundo em `forest-soft`
- texto `white`
- borda em `gold` suave ou `line-light`
- sem preto puro como base padrão

Hover:

- micro elevação (`-1px`)
- fundo em `forest`
- borda mais clara e perceptível
- manter contraste alto sem parecer brilho artificial

#### Button Motion Rules

- todos os botões clicáveis devem ter estado `hover`
- resposta visual mínima obrigatória:
  - `translateY(-1px)` ou equivalente
  - mudança sutil de fundo, borda ou sombra
- duração recomendada: `200ms` a `300ms`
- evitar:
  - zoom perceptível
  - bounce
  - glow chamativo
  - hover apenas por mudança quase imperceptível de opacidade
- componentes com o mesmo contexto visual devem reutilizar o mesmo padrão de botão; não criar um hover diferente para cada seção sem motivo funcional

### Inputs

- usar `Inter`
- forte legibilidade
- borda sutil
- sem excesso de sombra
- foco discreto porém claro

### Cards

Usar só quando realmente necessário.

Preferir:

- blocos editoriais
- imagens amplas
- painéis limpos

### Reviews

- mais tipografia que caixa
- layout arejado
- pouco chrome visual
- permitir composição com Google, Booking e Tripadvisor quando reforçar confiança sem parecer widget
- `EditorialStats` é o padrão obrigatório para esse tipo de composição visual

### Seasonal Campaigns

- `SpecialDatesSection` é o padrão oficial para datas especiais, feriados e campanhas sazonais da home
- deve reutilizar tokens semânticos do sistema (`primary`, `secondary`, `background`, `font-display`, `font-sans`)
- não recriar essa seção manualmente com outro grid, outra tipografia ou outra hierarquia visual
- se houver variação futura, ela deve nascer como evolução deste padrão, não como bloco paralelo

## 18. Implementation Tokens

### 18.1 Section Spacing Rules

O espaçamento vertical entre seções deve ser padronizado por tokens semânticos.

Nunca definir `py-*`, `pt-*` e `pb-*` de forma arbitrária em cada seção quando o caso de uso já existir abaixo.

Tokens oficiais:

- `section-space-sm`
  - uso: seções compactas de apoio editorial
  - mobile: `56px`
  - desktop: `80px`
- `section-space-md`
  - uso: padrão para seções principais de conteúdo
  - mobile: `64px`
  - desktop: `96px`
- `section-space-lg`
  - uso: blocos emocionais de fechamento ou grandes CTAs
  - mobile: `80px`
  - desktop: `112px`
- `section-space-overlap`
  - uso: seção logo abaixo do hero com widget sobreposto
  - mobile: `padding-top 80px`, `padding-bottom 64px`
  - desktop: `padding-top 96px`, `padding-bottom 96px`

Regras:

- `section-space-md` é o padrão da home
- não misturar espaçamentos diferentes entre seções equivalentes sem motivo funcional
- seções com fotografia e CTA final podem subir para `section-space-lg`
- blocos operacionais ou editoriais compactos podem usar `section-space-sm`
- quando houver sobreposição de widget com hero, usar `section-space-overlap`

Sugestão de tokens de marca:

```ts
export const brand = {
  forest: "#283223",
  gold: "#BBB863",
  cream: "#F5F5F5",
  white: "#FFFFFF",
  black: "#090909",
  whatsapp: "#00E676",
};

export const radius = {
  sm: "8px",
  md: "16px",
  lg: "24px",
};

export const spacing = {
  sectionSm: {
    mobile: "56px",
    desktop: "80px",
  },
  sectionMd: {
    mobile: "64px",
    desktop: "96px",
  },
  sectionLg: {
    mobile: "80px",
    desktop: "112px",
  },
  sectionOverlap: {
    paddingTopMobile: "80px",
    paddingBottomMobile: "64px",
    paddingTopDesktop: "96px",
    paddingBottomDesktop: "96px",
  },
};
```

Sugestão de tipografia:

```ts
export const typography = {
  display: "Libre Baskerville",
  ui: "Inter",
};
```

## 19. Final Design Statement

O site do Hotel Pousada Delplata deve parecer menos um website de hotel e mais uma experiência editorial de desejo com reserva embutida.

Ele precisa comunicar:

`descanso real, sofisticação natural e momentos em família em Serra Negra`

Se qualquer elemento parecer:

- portal
- dashboard
- sistema legado
- turismo genérico

ele está fora do sistema.
